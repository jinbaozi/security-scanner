# Phase 0: 发现阶段（Reconnaissance）

> 本文件指导 Recon Agent 执行目录探索、文件分类和分片规划。

## 执行时机

在 Phase -0（输入物化）完成后执行。由 Orchestrator 派发单个 Recon Agent 执行。

## 输入与输出

- 输入：Phase -0 生成的 materialization JSON，而不是原始包文件路径。普通目录输入也必须先被标记为 `source_tree` / `raw_directory`。
- 输出：Scan Plan JSON，包含 `materialization` 对象、物化源码根、二进制根、文件分类、分片、排除项和复杂度评估。

Recon 只能扫描以下根：

- `source_roots[].path`：来源为 `source_prepped` 或 `raw_directory`。
- `binary_roots[].path`：来源为 `binary_rpm`。

SRPM 外层 `Source*.tar.*`、`Patch*`、`.spec` 是物化证据，Source*.tar 不得当作已扫描源码；只有 `%prep` 后的 `source_prepped` 文件可以进入 `source_shards`。

## Step 1: 验证物化目标路径

```bash
if [ "{materialization.status}" != "ready" ]; then
  echo "ERROR: 输入物化未完成: {materialization.status}"
  exit 1
fi

for scan_root in {source_roots_and_binary_roots}; do
  if [ ! -d "$scan_root" ]; then
    echo "ERROR: 物化根目录不存在或不是目录: $scan_root"
    exit 1
  fi
done
```

文件数超过 10,000 时必须警告用户，建议缩小扫描范围或确认继续。

## Step 2: 目录探索

```bash
find {source_roots_and_binary_roots} -type f \
  -not -path "*/.git/*" \
  -not -path "*/.svn/*" \
  -not -path "*/.hg/*" \
  -not -path "*/node_modules/*" \
  -not -path "*/__pycache__/*" \
  -not -path "*/.venv/*" \
  -not -path "*/venv/*" \
  -not -path "*/.tox/*" \
  2>/dev/null | sort > /tmp/recon_all_files.txt
```

Recon 必须为每个文件保留 `origin` 标记：

- `source_prepped`：来自 SRPM `%prep` 后源码树。
- `binary_rpm`：来自 binary RPM 展开根。
- `raw_directory`：来自普通已展开目录，未验证 RPM patch 语义。

`all_files`、`elf_files`、`dependency_files` 同时覆盖 source roots 与 binary roots。`source_shards` 只允许使用 `source_prepped` 或 `raw_directory` 源码文件；binary root 中的源码样文件不得混入源码 shard，除非 Orchestrator 明确标记该 binary root 中包含源码交付物并记录原因。

## Step 3: 文件分类

### 3.1 第三方/生成代码

```bash
grep -E "(vendor/|third_party/|thirdparty/|external/|generated/|gen/|node_modules/|__pycache__/|\.git/|\.svn/|\.hg/|target/|build/|dist/|\.venv/|venv/)" /tmp/recon_all_files.txt > /tmp/recon_excluded_thirdparty.txt
grep -vE "(vendor/|third_party/|thirdparty/|external/|generated/|gen/|node_modules/|__pycache__/|\.git/|\.svn/|\.hg/|target/|build/|dist/|\.venv/|venv/)" /tmp/recon_all_files.txt > /tmp/recon_scan_files.txt
```

还需检查文件头部前 5 行，命中 `AUTO-GENERATED FILE`、`DO NOT EDIT`、`Code generated by`、`Automatically generated` 时排除。

### 3.2 ELF 文件

```bash
if which file >/dev/null 2>&1; then
  while IFS= read -r filepath; do
    if file "$filepath" 2>/dev/null | grep -q "ELF"; then
      echo "$filepath" >> /tmp/recon_elf_files.txt
    fi
  done < /tmp/recon_scan_files.txt
else
  while IFS= read -r filepath; do
    if which xxd >/dev/null 2>&1; then
      head -c 4 "$filepath" 2>/dev/null | xxd -p 2>/dev/null | grep -q "7f454c46" && echo "$filepath" >> /tmp/recon_elf_files.txt
    elif which od >/dev/null 2>&1; then
      head -c 4 "$filepath" 2>/dev/null | od -A n -t x1 2>/dev/null | grep -q "7f 45 4c 46" && echo "$filepath" >> /tmp/recon_elf_files.txt
    else
      head -c 4 "$filepath" 2>/dev/null | LC_ALL=C grep -q "$(printf '\177ELF')" && echo "$filepath" >> /tmp/recon_elf_files.txt
    fi
  done < /tmp/recon_scan_files.txt
fi
```

### 3.3 源码文件

```bash
grep -E "\.(c|cpp|cc|cxx|h|hpp|hxx|go|py|sh|bash|java|js|ts|rb|rs|pl|php|cs|kt|swift|m|mm)$" /tmp/recon_scan_files.txt > /tmp/recon_source_files.txt
```

### 3.4 配置文件

```bash
grep -E "\.(conf|cfg|ini|yaml|yml|json|toml|xml|properties|env|config|config\.go|config\.py)$" /tmp/recon_scan_files.txt > /tmp/recon_config_files.txt
```

配置文件也必须进入 URL Scanner 和 Secret Scanner。

### 3.5 未分类文件

未归入以上类别的文件写入 `excluded`，reason 为 `unclassified`。敏感文件扫描和权限扫描仍可消费完整文件列表。

### 3.6 依赖文件

```bash
grep -E "(^|/)(package\.json|package-lock\.json|requirements\.txt|Pipfile\.lock|go\.mod|go\.sum|pom\.xml|Cargo\.toml|vcpkg\.json|conanfile\.txt)$" /tmp/recon_scan_files.txt > /tmp/recon_dependency_files.txt
```

### 3.7 Docker 文件

```bash
grep -E "(^|/)(Dockerfile|docker-compose\.yml|docker-compose\.yaml)$" /tmp/recon_scan_files.txt > /tmp/recon_docker_files.txt
```

## Step 4: 规模评估与分片

```text
< 50     -> 不分片，每个维度 1 个 Scanner Agent
50-200   -> 按目录分片，每个维度 2-4 个 Agent
200-1000 -> 按文件类型+目录分片，每个维度 4-8 个 Agent
> 1000   -> 先排除更多第三方代码，再分片；总 Agent 数上限 16
```

分片算法：

1. 按目录分组。
2. 小目录（<50 文件）合并到同一分片。
3. 大目录（>=50 文件）拆分为多个分片。
4. 确保每个分片文件数不超过 50。

## Step 5: 生成 Scan Plan

```json
{
  "target": {
    "source_roots": ["/absolute/path/to/materialized/rpmbuild/pkg/BUILD/pkg-1"],
    "binary_roots": ["/absolute/path/to/materialized/binary-root/pkg.x86_64"]
  },
  "materialization": {
    "input_kind": "srpm|binary_rpm|package_directory|source_tree",
    "source_roots": [{"path": "/absolute/path/to/materialized/rpmbuild/pkg/BUILD/pkg-1", "origin": "source_prepped"}],
    "binary_roots": [{"path": "/absolute/path/to/materialized/binary-root/pkg.x86_64", "origin": "binary_rpm"}],
    "srpm_spec_files": ["/absolute/path/to/pkg.spec"],
    "applied_patches": ["fix.patch"],
    "builddep_attempted": false,
    "builddep_status": "not_required",
    "status": "ready|blocked",
    "errors": [],
    "audit_log": []
  },
  "component_name": "从目录名提取",
  "total_files": 1234,
  "scan_files": 1100,
  "elf_files": [{"path": "/path/to/binary1", "origin": "binary_rpm"}],
  "source_shards": [
    {
      "id": 0,
      "files": [{"path": "/path/to/file1.c", "origin": "source_prepped"}],
      "type": "source",
      "dir": "src/core"
    }
  ],
  "config_files": [{"path": "/path/to/config.yaml", "origin": "source_prepped"}],
  "dependency_files": [{"path": "/path/to/package.json", "origin": "source_prepped"}],
  "docker_files": [{"path": "/path/to/Dockerfile", "origin": "source_prepped"}],
  "crypto_relevant_files": [{"path": "/path/to/file1.c", "origin": "source_prepped"}],
  "network_relevant_files": [{"path": "/path/to/config.yaml", "origin": "source_prepped"}],
  "component_profile_relevant_files": [{"path": "/path/to/Dockerfile", "origin": "source_prepped"}],
  "all_files": [{"path": "/path/to/file1.c", "origin": "source_prepped"}],
  "excluded": [
    {"path": "/path/to/vendor/foo.go", "reason": "third_party:vendor"}
  ],
  "estimated_complexity": "small|medium|large|very_large",
  "timestamp": "ISO 8601 格式时间戳"
}
```

`dependency_files`: package.json, package-lock.json, requirements.txt, Pipfile.lock, go.mod, go.sum, pom.xml, Cargo.toml, vcpkg.json, conanfile.txt

`docker_files`: Dockerfile, docker-compose.yml, docker-compose.yaml

`crypto_relevant_files` 是 `source_files ∪ config_files` 按语言过滤（Tier-1 或 Tier-2）。

`network_relevant_files` 是 `source_files ∪ config_files` 过滤包含端口/协议模式的文件。

`component_profile_relevant_files` 是 `source_files ∪ config_files ∪ docker_files`。

`component_name` 取目标路径最后一级目录名。

当 `materialization.status=blocked` 时，Recon 不得伪造源码覆盖；应返回包含 materialization 错误的降级 Scan Plan，供 Reporter 输出阻断说明。

## 审计点 A0

Orchestrator 收到 Scan Plan 后验证：

1. 覆盖率：`elf_files + source_shards + config_files + excluded` 与 `total_files` 的差异不超过 5%。
2. 分片大小：每个 shard 的文件数不超过 50。
3. 目录完整性：对比 `find -type d` 输出，检查明显目录遗漏。

审计结果：

- PASS：进入 Phase 1。
- WARN：记录警告，继续进入 Phase 1。
- FAIL：重新运行 Recon Agent，最多重试 2 次；仍失败则降级输出。

## 异常处理

| 异常 | 处理 |
|------|------|
| 目标路径不存在 | 立即终止，输出错误信息 |
| 目录为空 | 终止扫描，提示用户检查路径 |
| 文件数超过 10,000 | 警告用户，建议缩小扫描范围或确认继续 |
| `file` 命令不可用 | 降级到 ELF magic bytes 检测 + 扩展名分类 |
| 某些目录无读取权限 | 跳过无权限目录，记录到 excluded（reason: `permission_denied`） |
| 分片后 Agent 数超过 16 | 合并分片，增大每片文件数上限并记录降级 |
